## Основные положения

### Базовый URL

```
https://api-developer.macdent.kz/<группа>/<метод>
```

Где:
- `<группа>` — название группы методов (например, `profile`, `patient`, `zapis`);
- `<метод>` — название конкретного метода внутри группы (например, `get`, `find`, `add`).

**Пример:**
```
https://api-developer.macdent.kz/profile/get
```

### Формат запросов

- Метод: `POST` или `GET`.
- Тип данных: `multipart/form-data`, обычный `POST`, либо JSON-строка.
- При передаче более **2 КБ** данных — обязательно использовать `POST`.
- Параметр `access_token` необходимо передавать с каждым запросом (кроме публичных методов).

**Пример запроса:**
```
POST https://api-developer.macdent.kz/test/test?access_token=43543534n34nhb353124njnjsz
```

### Формат ответа

Сервер всегда возвращает JSON со следующими обязательными полями:

| Поле | Тип | Описание |
|------|-----|----------|
| `response` | `0` или `1` | `1` — запрос успешен, `0` — ошибка |
| `error` | string | Описание ошибки (только при `response: 0`) |
| `isTokenNeedToBeUpdated` | `0` или `1` | `1` — токен устарел, требуется переавторизация |

## Авторизация

Все методы, кроме публичных, требуют `access_token`.

- Ключ выдаётся службой поддержки **Macdent**.
- Срок действия ключа **не ограничен**.
- Ключ может быть обнулён по запросу в службу поддержки.
- Ключ предназначен для интеграции с внешними системами.

Если токен недействителен, ответ будет содержать:
```json
{
  "response": 0,
  "error": "Описание ошибки",
  "isTokenNeedToBeUpdated": 1
}
```

В этом случае необходимо получить новый `access_token` и повторить запрос.

## Обработка ошибок

| Код `response` | Значение |
|----------------|----------|
| `1` | Запрос выполнен корректно |
| `0` | Запрос выполнен с ошибкой — см. поле `error` |

Если `isTokenNeedToBeUpdated = 1` — сессия устарела, нужна переавторизация.

## Webhook

Система поддерживает отправку уведомлений об изменении объектов на ваш URL.

### Поддерживаемые события

- `onCreate` — создание объекта;
- `onChange` — изменение объекта;
- `onRemove` — удаление объекта.

### Методы управления Webhook

| Метод | Назначение |
|-------|------------|
| `webhook/set` | Установить URL для получения событий |
| `webhook/get` | Получить текущий URL и статус вебхука |
| `webhook/send_debug` | Имитировать отправку событий и посмотреть ответ вашей системы |

### Логика повторных отправок

- Если ваш сервер ответил кодом **200** — попытки прекращаются.
- Если код ответа **отличен от 200** — Macdent повторяет отправку **каждые 2 минуты**.
- После **20 неудачных попыток** — отправка прекращается.

## Real-Time звонки

Для корректной работы со звонками в режиме реального времени соблюдайте следующий порядок вызовов:

| Шаг | Метод | Когда вызывать |
|-----|-------|----------------|
| 1 | `call/incoming` | Пришёл входящий звонок |
| 2 | `call/ofhook` | Пользователь взял трубку / начался исходящий звонок |
| 3 | `call/end` | Звонок завершён |
| 4 | `call/send_recording` | Отправка записи звонка в журнал |

## Примеры ответов

### Успешный ответ

```json
{
  "response": 1,
  "data": {
    "id": 12345,
    "name": "Иванов Иван"
  }
}
```

### Ошибка

```json
{
  "response": 0,
  "error": "Пациент не найден"
}
```

### Токен устарел

```json
{
  "response": 0,
  "error": "Token is invalid",
  "isTokenNeedToBeUpdated": 1
}
```

## Краткий чек-лист для интеграции

1. Получить `access_token` у службы поддержки Macdent.
2. Передавать `access_token` с каждым запросом.
3. Использовать `POST` для данных свыше 2 КБ.
4. Всегда проверять поле `response` в ответе.
5. При `isTokenNeedToBeUpdated: 1` — получить новый токен.
6. Настроить webhook через `webhook/set` и обязательно возвращать **HTTP 200** на принятые события.
7. Для звонков соблюдать порядок: `incoming` → `ofhook` → `end` → `send_recording`.
